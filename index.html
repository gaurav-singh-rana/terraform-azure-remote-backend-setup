🚀 Terraform Azure Remote Backend Setup

Production-style Terraform setup using an existing Azure Storage Account for remote state management.

Terraform Azure DevOps
----

📌 What This Project Does

This project demonstrates how to:

  • Configure Terraform remote backend in Azure
  • Use pre-created Storage Account and Blob Container
  • Manage Terraform state securely
  • Use implicit dependency between resources
---

⚠️ Important Concept

Terraform backend cannot be created and used in the same run.

That’s why this project assumes:

  • Storage Account already exists
  • Container already exists
  • Backend is configured using those resources
---

🧱 Resources in This Project

  • Resource Group
  • Storage Account (for tfstate)
  • Blob Container
---

⚙️ Terraform Code

resource "azurerm_resource_group" "rg_dev_main" { name = var.rg_name location = var.rg_location } resource "azurerm_storage_account" "sa_tfstate" { name = "sttfstateci001" resource_group_name = azurerm_resource_group.rg_dev_main.name location = azurerm_resource_group.rg_dev_main.location account_tier = "Standard" account_replication_type = "GRS" } resource "azurerm_storage_container" "sc_tfstate" { name = "tfstate" storage_account_id = azurerm_storage_account.sa_tfstate.id container_access_type = "private" }
---

🔐 Backend Configuration

terraform { backend "azurerm" { resource_group_name = "rg_dev" storage_account_name = "rgdevsa" container_name = "rgdevsecure" key = "terraform.tfstate" } }
---

🚀 How to Run

terraform init terraform plan terraform apply
---

🧠 Key Learnings

  • Terraform backend lifecycle
  • Remote state management
  • Implicit resource dependencies
  • Azure infrastructure provisioning
---

❌ Common Mistake

backend "azurerm" { resource_group_name = azurerm_resource_group.rg.name ❌ }

Backend cannot reference Terraform resources.
